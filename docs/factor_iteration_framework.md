# 因子迭代框架

本文档回答一个更具体的问题：

- 在现有基金研究框架下，如何持续迭代“主候选评分体系”里的因子集合与权重

它不替代：

- [`factor_catalog.md`](/Users/liupeng/.codex/projects/fund_research_v2/docs/factor_catalog.md)：回答“因子是什么意思、当前哪些配置在用”
- [`baseline_upgrade_checklist.md`](/Users/liupeng/.codex/projects/fund_research_v2/docs/baseline_upgrade_checklist.md)：回答“候选评分体系能否升级为默认 baseline”

本文档只关心一件事：

- 因子在进入主候选评分体系之前，应如何做去留决策、竞争位比较和小步迭代

## 1. 使用场景

当你处于下面这些阶段时，应优先使用本文档：

- 还在决定哪些因子应该进入主候选评分体系
- 还在比较某个新因子是否能替换旧因子
- 还在收敛每个因子类别内部的职责边界
- 还不适合直接讨论“是否升级默认 baseline”

如果当前已经完成：

- 候选配置完整实验
- 实验对比
- 稳健性分析
- 候选补证

且当前要回答的是“要不要升 baseline”，则应切换到：

- [`baseline_upgrade_checklist.md`](/Users/liupeng/.codex/projects/fund_research_v2/docs/baseline_upgrade_checklist.md)

## 2. 因子管理分三层

为了避免评分体系不断膨胀，所有因子都必须落在以下三层之一：

### 2.1 主候选层

含义：

- 直接进入当前主候选评分体系
- 参与正式 A/B 实验与稳健性分析

进入条件：

- 金融逻辑清楚
- 单因子证据基本过关
- 与已有主因子不高度重复
- 放进组合后能带来清晰边际贡献

### 2.2 观察层

含义：

- 保留计算与跟踪
- 但暂不进入当前主候选评分体系

典型场景：

- 金融逻辑成立，但组合层证据还不够
- 对现有主因子可能有替代潜力，但还需要更多样本验证
- 信息可能有价值，但太短周期、太事件驱动或阶段敏感

### 2.3 淘汰层

含义：

- 当前不建议继续投入精力
- 除非样本规模、业务目标或口径发生明显变化，否则不再优先考虑

典型原因：

- 方向不稳定
- 与已有主因子高度重复
- 组合层收益改善不成立
- 主要依赖短样本、事件样本或特殊阶段

## 3. 选因子的四个判断维度

任何因子想进入主候选层，至少要同时过下面四关。

### 3.1 金融语义是否清楚

要先能回答：

- 这个因子到底在衡量什么
- 它为什么可能与未来持有体验或主动管理能力相关

如果一个因子只能回答：

- “历史上它好像有用”

但说不清：

- 它在基金业务上到底对应什么行为或风险

则不应优先进入主候选层。

### 3.2 单因子证据是否过关

至少要观察：

- RankIC 是否长期方向一致
- top-bottom spread 是否有基本方向性
- 不同年份或样本扩容后是否大幅翻向

这里不要求因子在所有年份都完美稳定，但至少不能表现为：

- 一扩样本就失效
- 只在极少数窗口有效
- 方向经常反转

### 3.3 是否与已有主因子重复

很多因子表面不同，本质上抓的是同一件事。

常见重复关系：

- `ret_12m` 与 `excess_ret_12m`
- `vol_12m`、`downside_vol_12m`、`max_drawdown_12m`
- 多个短周期超额因子之间

如果一个新因子只是用另一种形式重复已有主因子的暴露，则通常不应同时重仓。

### 3.4 进入组合后是否真有边际贡献

最终不是看单因子报表，而是看它进入评分体系以后：

- 累计收益是否改善
- 超额收益是否改善
- 回撤是否恶化
- 波动与换手是否失控
- 优势是否只集中在少数月份

只有单因子证据，没有组合边际贡献，不足以进入主候选层。

## 4. 不同因子类别的职责边界

建议先把因子分到以下三类，再在每类内部做竞争。

### 4.1 收益质量类

目标：

- 判断基金是否具备持续创造收益或超额收益的能力

典型因子：

- `ret_12m`
- `excess_ret_12m`
- `excess_hit_rate_12m`
- `excess_consistency_12m`

### 4.2 风险控制类

目标：

- 判断收益是否建立在不可接受的波动、回撤或左尾风险之上

典型因子：

- `downside_vol_12m`
- `worst_3m_avg_return_12m`
- `vol_12m`
- `max_drawdown_12m`

### 4.3 稳定性与可投性类

目标：

- 判断基金是否足够稳定、可持续、适合作为研究对象

典型因子：

- `asset_stability_12m`
- `manager_tenure_months`
- 其他规模、流动性、组织稳定性相关因子

## 5. 主候选体系的推荐结构

为了防止因子堆叠，建议主候选评分体系先采用小而清楚的结构：

- 收益质量类：`1` 到 `2` 个主因子
- 风险控制类：`1` 到 `2` 个主因子
- 稳定性类：`1` 个辅助因子

当前推荐的基础模板是：

- 收益质量：`2`
- 风险控制：`2`
- 稳定性：`1`

也就是典型的 `2 + 2 + 1` 结构。

这个结构的优点：

- 每类职责清楚
- 不容易重复加权
- A/B 结果更容易解释

- `高`
  - 金融语义清楚

- `中`
  - 金融语义成立

## 7. 当前推荐的主候选结构

当前更推荐的主候选结构接近 `v2-lite`：

- 收益质量主干：
  - `excess_ret_12m`
  - `ret_12m`
- 风险控制主干：
  - `downside_vol_12m`
  - `worst_3m_avg_return_12m`
- 稳定性辅助：
  - `asset_stability_12m`

这套结构的出发点不是“把最强因子全堆进去”，而是：

- 每个位置都只承担一类清楚职责
- 避免短周期和事件型因子把体系带偏
- 让组合层结果更容易解释

- 主要挑战者：`manager_tenure_months`
- 结论：当前保留 `asset_stability_12m`

## 9. 单轮迭代只允许三种动作

为了让每轮实验结果可解释，建议每轮只做下面三种动作之一：

### 9.1 `replace`

含义：

- 用新因子替换一个旧因子

示例：

- `excess_hit_rate_12m` 替换 `ret_12m`

优点：

- 最容易解释
- 最适合做一轮一轮的结构收敛

### 9.2 `add`

含义：

- 引入一个新因子，但总权重不变

示例：

- 从稳定性类切出一部分权重给新的观察层因子

注意：

- 只有在已经确认新因子与现有因子不高度重复时才建议这样做

### 9.3 `reweight`

含义：

- 不换因子，只调权重

示例：

- 把 `excess_ret_12m` 权重从 `0.7` 调到 `0.6`

注意：

- 只有在结构已经相对稳定时，才建议进入大规模调权重阶段

## 10. 单轮实验必须回答的 6 个问题

每一轮因子迭代都必须回答以下六个问题：

1. 组合累计收益有没有改善
2. 超额收益有没有改善
3. 回撤有没有恶化
4. 波动和换手有没有明显变差
5. 优势是不是只集中在少数月份
6. 扩样本后改进还在不在

如果一轮实验只能回答：

- “累计收益更高了”

但回答不了其余问题，则不应推进到主候选层。

## 11. 因子去留的机械化规则

建议后续按下面规则机械化执行。

### 11.1 升到主候选层

需要同时满足：

- 金融解释清楚
- 单因子方向基本稳定
- 与现有主因子不高度重复
- 所在方案相对当前主候选：
  - 超额收益提升
  - 回撤未显著恶化
  - `return_concentration_flag = 0`
  - `time_slice_consistency_flag = 1`

### 11.2 留在观察层

适用场景：

- 单因子有潜力
- 但组合层证据还不够
- 或样本扩容后优势明显收敛

### 11.3 进入淘汰层

适用场景：

- 方向反复
- 与已有主因子高度重复
- 进入组合后只抬波动、回撤或换手
- 明显依赖小样本、短样本或事件样本

## 16. 一句话原则

评分体系迭代的目标，不是不断增加因子数量，而是把现有因子体系收敛成：

- 结构清楚
- 职责明确
- 风险可解释
- 能小步迭代

如果某个因子只是“看起来可能有用”，但无法明确它在体系中的位置和职责，就不应急于纳入主候选评分体系。
