# 系统架构说明

## 1. 项目定位

`fund_research_v2` 是一个面向中国市场场外公募主动权益基金的月频量化研究平台。

当前版本定位为：

- 研究系统，不是实盘交易系统
- 可复现、可测试、可审计的研究内核
- 用配置和数据快照驱动实验，而不是手工修改脚本

当前版本不包含：

- 实盘申赎执行
- 高频或日频策略
- 完整 Web 平台
- 黑盒优化器或机器学习选基模型

## 2. 总体链路

系统当前的标准流程如下：

1. `fetch`
   从 `sample` 或 `tushare` 数据源生成/拉取原始研究数据，并落盘到 `data/raw/<data_source>`
2. `build-universe`
   按月构建基金池，输出每只基金在每个月是否可投资，以及剔除原因
3. `compute-features`
   对基金池上的基金计算月频特征
4. `run-ranking`
   对同月基金做横截面打分和排序
5. `run-backtest`
   使用“当月信号、下一月执行”的口径做历史回测
6. `run-experiment`
   执行完整闭环，并输出报告、实验记录与结果文件

## 3. 分层设计

### 3.1 `common`

负责通用基础设施：

- 配置加载与校验
- 数据契约定义
- 时间工具
- IO 工具
- 工作流编排

### 3.2 `data_ingestion`

负责数据接入：

- `sample` 样例数据生成
- `tushare` 原始数据拉取
- 原始层缓存读写
- 原始字段到内部契约的初步映射

### 3.3 `universe`

负责基金池构建：

- 基金类型过滤
- 名称关键词排除
- 历史长度要求
- 基金成立时间要求
- 规模门槛要求
- 剔除原因审计

### 3.4 `features`

负责月频特征计算：

- 3/6/12 月收益
- 12 月超基准收益
- 12 月波动率
- 12 月下行波动率
- 12 月最大回撤
- 经理任期月数
- 规模稳定性

### 3.5 `ranking`

负责月度横截面评分：

- 仅对同月可投资基金排序
- 按字段横截面归一化
- 汇总为三类因子分：
  - 收益质量
  - 风险控制
  - 稳定性质量
- 使用配置中的权重合成总分

### 3.6 `portfolio`

负责组合构建：

- 以正式最新研究月评分结果为输入
- 默认按总分排序选前 `N`
- 默认等权
- 控制单基金权重上限
- 控制单基金公司入选数量上限

### 3.7 `backtest`

负责历史回测：

- 使用 `signal_month -> execution_month` 的月频口径
- 组合在信号月末生成
- 下一月收益作为执行结果
- 按换手收取固定交易成本

### 3.8 `evaluation`

负责绩效汇总：

- 累计收益
- 年化收益
- 年化波动
- 最大回撤
- 胜率
- 基准累计收益
- 超额累计收益

### 3.9 `reporting`

负责产出报告：

- 回测报告
- 实验报告

## 4. 当前默认设计选择

为了符合 `AGENTS.md` 的约定，当前版本明确采用以下默认口径：

- 基金范围：场外公募主动权益基金
- 调仓频率：月频
- 信号时点：最后一个完整自然月的月末
- 执行时点：下一月
- 组合方法：规则约束法
- 特征层 benchmark：使用 `benchmark_monthly` 中的 `benchmark_return_1m`
- 主回测 benchmark：固定使用 `benchmark.default_key` 对应的市场 benchmark
- 成本：固定 `transaction_cost_bps`
- A/C 份额：默认按基金实体合并，不把份额差异当作独立 alpha 来源

这里的“最后一个完整自然月”含义是：

- 若 `as_of_date` 仍处于月中，则正式最新研究月自动回退到上一个月
- 当月 raw 数据即使已经部分到达，也只能作为观察快照，不进入正式组合与正式审计
